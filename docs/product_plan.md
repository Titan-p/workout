# 产品计划

## 产品方向

- 个人自用
- Excel 优先
- Kdocs 同步作为计划维护入口
- 训练记录、计划维护、历史复盘围绕同一份计划数据展开

## 功能优先级

### 1. 训练闭环

目标：把训练前、训练中、训练后记录闭合到同一条链路里。

当前基础：

- 训练页已经提供今日计划、开始训练、上传计划、历史记录入口
- 训练 API 已经跑通训练会话和训练组记录流程
- `training_sessions` / `training_sets` 已经具备 `rpe`、`notes`、`metadata` 的承载位

下一步功能：

- 训练中记录 `RPE`
- 训练中记录训练时长
- 训练中记录体重
- 训练中记录疲劳 / 疼痛状态
- 训练结束填写总结
- 单组编辑
- 撤销最近一次记录

### 2. 复盘分析

目标：把历史记录升级成能持续复盘的分析面板。

当前基础：

- `/api/training-history` 已经返回最近训练组列表
- 组合动作已经支持 `components` 识别

下一步功能：

- 动作趋势
- 最近 PR
- 周训练量
- 计划完成率
- 阶段完成率
- 月历视图
- 组合动作拆分统计

### 3. 计划同步

目标：把计划导入从一次性导表，升级成可预览、可比对、可回滚的维护流程。

当前基础：

- 页面已经提供 Excel 上传入口
- 仓库已经提供 Kdocs 同步脚本
- 当前导入逻辑会清空 `workout_plans` 并导入最新阶段

下一步功能：

- Kdocs 一键同步
- 导入预览
- 差异确认
- 版本回滚

### 4. 移动体验

目标：把周视图和训练页继续压缩到更适合手机连续使用的节奏。

当前基础：

- 页面已经具备移动端布局
- 已有休息计时
- 已有本地缓存

下一步功能：

- Manifest
- Service Worker
- 离线补同步
- 休息结束声音提醒
- 休息结束振动提醒
- 训练中快捷输入按钮

### 5. 负荷管理

目标：把 `session RPE × Duration` 变成可长期监控的周负荷面板。

当前基础：

- 训练页已经支持 `RPE + 训练总结`
- `training_sessions` 已经具备 session 级承载位
- Excel 中已经存在 `Load Monitor` sheet 作为口径参考

下一步功能：

- 结束训练记录 `session RPE`
- 结束训练记录 `duration`
- 结束训练记录 `训练时段`
- 每日记录 `体重 / 疲劳 / 疼痛`
- `Load Monitor` 周视图
- 周总负荷 / Chronic Load / ACWR

## 下一轮切片

下一轮优先实现四项：

1. `RPE + 训练总结`
2. `动作趋势页`
3. `Kdocs 一键同步`
4. `导入版本回滚`
5. `Load Monitor 周视图`

## 预期收益

- 提升日常训练记录效率
- 提升长期数据复盘价值
- 降低计划维护成本
- 提升手机端使用顺手程度
